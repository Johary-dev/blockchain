Vote décentralisé

Interface HTML pour le mini-projet Blockchain : propositions on-chain, un vote par adresse et clôture après deadline.

Interaction avec le contrat

Ce fichier HTML sert de front-end demandé dans le PDF. Il doit permettre à un utilisateur de se connecter avec MetaMask, lire l'état du smart contract, voter pour une proposition et déclencher les fonctions publiques utiles.

Ce qu'on doit voir dans le HTML

  • Un bouton de connexion MetaMask.
  • Un champ pour l'adresse du contrat Sepolia.
  • La liste des propositions lues depuis la blockchain.
  • Un bouton de vote par proposition.
  • Les compteurs de votes et le statut ouvert/fermé.
  • Une zone d'erreur ou de confirmation pour les transactions.
Pour la soutenance, remplacez le champ vide par l'adresse réelle du contrat déployé, puis montrez MetaMask, le vote et la mise à jour des résultats.

Propositions

Aucune donnée chargée. Connectez MetaMask, saisissez l'adresse du contrat, puis cliquez sur Charger les données.

Explication technique

Le contrat DecentralizedVoting expose les fonctions nécessaires au front : getAllProposals() pour lire les propositions, vote(id) pour voter, isVotingOpen() et getRemainingTime() pour afficher l'état, puis getWinner() et closeVoting() pour la fin du scrutin.

Cette page utilise ethers.js depuis un CDN. Elle peut donc être ouverte directement dans un navigateur, mais les transactions réelles nécessitent MetaMask, le réseau Sepolia et l'adresse du contrat déployé.
